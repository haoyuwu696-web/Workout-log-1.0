健身记录 App 开发代办事项（基于当前 7 个 HTML 原型）

项目目标
- 将现有原型页面（index/calories/timer/history/workout/body/settings）落地为可用的健身记录应用。
- 先完成前端可用版本（本地数据持久化），再预留后端扩展能力。

====================================
一、项目初始化与结构整理
====================================
1. 技术方案确认
   - [ ] 确定实现方式：纯 HTML/CSS/JS 增强，或迁移到框架（如 Vue/React）。
   - [ ] 确定是否保留 Tailwind CDN，或改为本地构建（便于维护与性能优化）。
   - [ ] 确定图表方案（继续使用 ECharts）。

2. 目录与资源规范
   - [ ] 规划目录结构：/pages /assets /styles /scripts /data。
   - [ ] 拆分公共资源（颜色、字体、通用组件样式）。
   - [ ] 统一命名规范（文件名、类名、数据字段名）。

3. 开发基础配置
   - [ ] 增加本地开发启动方式（如简单静态服务器）。
   - [ ] 增加代码格式化约定（Prettier 或统一手工规范）。
   - [ ] 增加版本迭代说明（changelog 习惯）。

====================================
二、核心数据模型设计（先于页面联动）
====================================
1. 用户与身体数据
   - [ ] 设计用户基础数据结构：身高、体重、年龄、性别、训练目标。
   - [ ] 设计身体指标结构：体脂率、围度、BMI、静息心率等（可选字段）。
   - [ ] 定义历史体测记录格式（按日期存储，支持回溯）。

2. 训练记录数据
   - [ ] 设计训练模板结构：动作、组数、次数、重量、休息时长。
   - [ ] 设计单次训练日志结构：开始时间、结束时间、总时长、总容量、备注。
   - [ ] 设计动作库结构：按肌群分类，支持新增自定义动作。

3. 饮食与热量数据
   - [ ] 设计食物条目结构：名称、份量、热量、蛋白质/碳水/脂肪（可选）。
   - [ ] 设计每日热量汇总结构：摄入、消耗、差值、目标。
   - [ ] 设计估算规则（基础代谢 + 活动系数 + 训练消耗）。

4. 通用数据策略
   - [ ] 定义本地存储 key 规范（localStorage / IndexedDB）。
   - [ ] 设计数据迁移机制（字段升级时兼容旧数据）。
   - [ ] 设计导入导出格式（JSON 备份恢复）。

====================================
三、7 个页面功能实现清单
====================================
A. 数据看板（index.html）
- [ ] 接入真实数据，替换当前示例数值。
- [ ] 完成“今日状态”计算逻辑（活动环、锻炼环进度）。
- [ ] 完成力量趋势图联动（按动作切换数据）。
- [ ] 完成体重趋势图联动（按周/月切换可扩展）。
- [ ] 完成“最近活动”列表与训练日志同步。

B. 训练记录（workout.html）
- [ ] 创建“新训练”流程（选择模板/空白开始）。
- [ ] 实现动作增删改（组数、重量、次数、休息）。
- [ ] 实现训练中实时统计（总容量、已训练时长）。
- [ ] 训练完成后写入历史记录。
- [ ] 增加训练备注与主观强度（RPE，可选）。

C. 计时器（timer.html）
- [ ] 实现基础计时（开始/暂停/继续/重置）。
- [ ] 实现组间倒计时（可配置秒数）。
- [ ] 增加提示方式（声音/震动占位）。
- [ ] 与训练页联动（自动启动休息计时可选）。

D. 卡路里估算（calories.html）
- [ ] 实现 TDEE/BMR 估算器。
- [ ] 支持增肌/减脂/维持目标下的建议热量。
- [ ] 支持录入每日饮食条目并自动汇总。
- [ ] 展示热量摄入与消耗差值。

E. 历史查看（history.html）
- [ ] 实现按日期查看训练记录。
- [ ] 增加筛选（动作/肌群/训练类型）。
- [ ] 增加统计视图（周频率、月容量）。
- [ ] 支持查看单次训练详情。

F. 身体数据（body.html）
- [ ] 实现体重、体脂、围度录入。
- [ ] 实现趋势图展示（周/月）。
- [ ] 提供目标区间提示（如体脂目标范围）。
- [ ] 与首页摘要联动显示关键变化。

G. 设置（settings.html）
- [ ] 用户资料设置（单位制、目标、提醒时间）。
- [ ] 数据管理（导出、导入、清空确认）。
- [ ] 应用偏好（主题、默认首页、提示开关）。
- [ ] 关于与版本信息。
