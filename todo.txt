健身记录 App 开发代办事项（基于当前 7 个 HTML 原型）

项目目标
- 将现有原型页面（index/calories/timer/history/workout/body/settings）落地为可用的健身记录应用。
- 先完成前端可用版本（本地数据持久化），再预留后端扩展能力。

====================================
一、项目初始化与结构整理（已明确方案 + 可执行落地）
====================================
1. 技术方案确认（已完成）
   - [x] 实现方式：采用纯 HTML/CSS/JS 增强，不迁移 Vue/React。
   - [x] 样式方案：由 Tailwind CDN 改为本地构建（便于维护、性能优化、离线可用）。
   - [x] 图表方案：继续使用 ECharts，并封装公共图表配置。

2. 目录与资源规范（本阶段完成）
   - [x] 目标目录结构确定：
         /
         ├─ pages/            # 7个页面（index/workout/timer/history/body/calories/settings）
         ├─ assets/
         │  ├─ icons/         # 图标与应用图片
         │  └─ fonts/         # 字体（如后续需要）
         ├─ styles/
         │  ├─ tailwind.css   # Tailwind 输入文件
         │  ├─ base.css       # 全局重置与基础样式
         │  ├─ components.css # 通用组件样式（卡片、按钮、表单）
         │  └─ utilities.css  # 自定义工具类
         ├─ scripts/
         │  ├─ core/          # storage、date、calc 等核心逻辑
         │  ├─ charts/        # chart-utils 与各页面图表配置
         │  └─ pages/         # 每个页面独立脚本
         ├─ data/             # mock 或初始化数据
         ├─ package.json
         ├─ tailwind.config.js
         └─ postcss.config.js

   - [x] 公共资源拆分规则：
         - 颜色、间距、圆角、阴影等设计令牌写入 tailwind.config.js。
         - 页面共用卡片/按钮/输入框样式集中在 components.css。
         - 图表通用主题（字体、网格、tooltip、配色）放到 scripts/charts/chart-utils.js。

   - [x] 命名规范（统一执行）：
         - 文件名：kebab-case（如 workout-history.js）。
         - CSS 类名：语义化 + kebab-case（如 stats-card、tab-item-active）。
         - JS 变量/函数：camelCase；常量：UPPER_SNAKE_CASE。
         - localStorage key：fitlog:{module}:{name}（如 fitlog:body:records）。
         - 数据字段：英文 camelCase（如 workoutDuration、totalVolume）。

3. 开发基础配置（本阶段完成）
   - [x] 本地开发启动方式：
         - npm scripts 约定：
           - dev：并行启动 Tailwind 监听 + 静态服务器。
           - build:css：构建压缩后的样式文件。
           - preview：预览生产构建结果。
         - 推荐开发服务：vite（仅作静态页面托管与热更新）或 live-server。

   - [x] 代码格式化约定：
         - 使用 Prettier 统一 HTML/CSS/JS 格式。
         - 统一 2 空格缩进，单引号，末尾分号，行宽 100。
         - 提交前执行格式化，避免多人协作样式漂移。

   - [x] 版本迭代说明（changelog 习惯）：
         - 版本号建议：0.x.y（原型到可用阶段）。
         - 记录模板：日期 / 版本 / 新增 / 优化 / 修复 / 数据变更说明。
         - 里程碑对齐：M1、M2、M3 每阶段结束产出一次变更记录。

4. 一阶段交付物检查清单（新增）
   - [ ] 建立 package.json、tailwind.config.js、postcss.config.js。
   - [ ] 完成 styles 与 scripts 基础目录落地。
   - [ ] 将 7 个页面迁入 pages/ 并修正资源相对路径。
   - [ ] 产出首版 chart-utils.js（全局图表主题 + resize 处理）。
   - [ ] 新增 .prettierrc 并完成全项目首次格式化。
